# Medarot-P
- Medarot-Pは、ゲームシリーズ「メダロット」のバトルシステム(ロボトル)をPythonで再現する試みです。
- ライブラリとしてPygameを利用

# 開発方針
- シンプルであること。パフォーマンス向上のために行数が増えるような最適化は避ける。
- ECSアーキテクチャの原則に従うこと。OOP的アプローチは避ける。
- 責務分離の徹底と、簡潔な依存方向を実現するディレクトリ構造を維持すること。

# 戦闘システム
- 行動選択後にチャージフェーズ、行動実行後にクールダウンフェーズを持つ、変則的なATB
- リアルタイムでゲージが溜まり、行動順が決まる
- ウェイト式ATBのため、メッセージ表示中は時間が停止する
- ゲージ進行状況は、アイコンの位置変化で表現される
- パーツ別HPシステム: メダロットのパーツ（頭部、右腕、左腕、脚部）ごとにHPを管理

# ゲームルール
- 勝利条件: 相手チームの全メダロットを機能停止させる（=頭部破壊）
- 攻撃時に使用するパーツを選択可能。破壊されたパーツは使用不可
- プレイヤーチーム（3体） vs エネミーチーム（3体）
